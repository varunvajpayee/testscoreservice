E) RISK ADJUSTED PERFORMANCE MEASURE

 

 

1.     Ex-Post Alpha (Jensen's Ex Post Alpha) - Systematic Risk Measure

 

 

It's the difference between actual return and the expected return based on Security market line. Since SML uses beta as systematic risk measure.

 

Ex Post Alpha = Rp - Rep

 

 

 

Rp = Actual portfolio return

Rep = Expected portfolio return as per the SML line.

P = Portfolio

M= Market portfolio

Rf= Risk free rate

 

 

2.     Treynor Measure (Alpha relative to Beta) - Systematic Risk Measure

 

A portfolio with positive alpha will have high treynor than market portfolio's treynor.  Treynor is the slope of risk free rate (rf) to P (portfolio).

 

 

Treynor= Rp-Rf/βp

 

 

 

3.     Sharpe Ratio - Total Risk Measure

 Slope of CML and calculated using standard deviation (total risk). It is useful if returns are normally distributed.

 

Sharpe Ratio= Rp-Rf/𝝈p

 

 

 

 

 

 

4.     The M2 (Modigliani & Modigliani) - Total Risk Measure

 

It is the value add or lost relative to the market if the portfolio has the same risk as the market. if portfolio risk is less than the market risk (standard deviation of Market portfolio) then borrow at RFR and invest in market or if portfolio risk is more risk than market then lends money at the risk-free rate.

 

 

 

5.     The information ratio

Like Sharp ratio, it's measured against the benchmark (instead of risk free rate). Excess return (return over benchmark) is also called active return. Standard deviation of active return is called active risk.

 

Information Ratio = Active Return/Active Risk

 

 

 Active Return = Rp - Rb

 

Active risk = 𝝈 (p-b)

 

 

6.     Quality control charts

 

Two Assumptions:

1.       Managers returns are independent from the period, normally distributed and their mean value is zero.

2.       Investment process doesn't change from period to period so variability should be constant.

 

Null hypothesis, H0, is that Managers value-added returns = 0 (manager's return = benchmark return),

Return above 0 line shows excess return (H0>0) so manager has superior capability to generate excess return.

 

 Type I and Type II errors:

A. Type I error  keeping bad managers (rejecting the null hypothesis when it is true; manager's return = benchmark's return). 

B. Type II error  Firing good managers (Fail to reject null when it is false).
